Project Overview

The Hospital Triage App was developed as Assignment #3 in my web development course. This full-stack application simulates an emergency room triage system, allowing medical staff to register patients, assign priority levels based on severity, and manage waiting lists in real-time. The system ensures critical patients receive immediate attention while efficiently managing hospital resources.

Technical Documentation

Core Technologies Used

  • Backend: Node.js with Express.js framework
  • Database: PostgreSQL with sophisticated relational schema
  • Frontend: HTML5, CSS3, JavaScript (ES6+)
  • API Design: RESTful architecture with JSON data exchange
  • Testing: Postman for API endpoint validation
  • Version Control: Git/GitHub with structured commit history

Database Schema & Design

The application uses a normalized relational database structure with proper table relationships:

  • Patients Table: Stores patient demographics and contact information
  • Triage_Records Table: Manages triage assessments with timestamps
  • Priority_Levels Table: Reference table for triage categories (1-5 scale)
  • Medical_Staff Table: Hospital personnel credentials and roles
-- Patients Table
CREATE TABLE patients (
    patient_id SERIAL PRIMARY KEY,
    code VARCHAR(50) NOT NULL,
    name VARCHAR(50) NOT NULL,
    injury_type varchar(50) NOT NULL,
    pain_level int,
    arrival_time timestamp(),
    priority_id int(),
);

Key Features & Implementation

1. Asynchronous Operations

Implemented JavaScript's async/await pattern with Fetch API for seamless patient data submissions and real-time updates without page reloads, maintaining optimal user experience. 

// Example: Async function to submit patient data
async function submitPatientData(patientData) {
    try {
        const response = await fetch('/api/patients', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(patientData)
        });
        
        if (!response.ok) throw new Error('Network response error');
        const result = await response.json();
        return result;
    } catch (error) {
        console.error('Error submitting patient data:', error);
        throw error;
    }
}

2. Complex Data Handling

Developed robust backend using Node.js with PostgreSQL, implementing complex queries with proper JOIN operations and transaction management to ensure data integrity across related tables.

3. Server-Client Interaction

Utilized Express.js to construct RESTful APIs with proper HTTP methods (GET, POST, PUT, DELETE) ensuring efficient and secure data transfer between frontend and backend services.

4. Testing Strategy

Implemented comprehensive testing using Postman for API validation, ensuring all endpoints respond correctly with appropriate status codes and data formats.

Challenges & Solutions

Challenge 1: Real-time Priority Updates

Problem: Ensuring the waiting list automatically reorders when new high-priority patients arrive.
Solution: Implemented database triggers and frontend polling mechanism to refresh the queue every 30 seconds.

Challenge 2: Data Validation Across Layers

Problem: Maintaining consistent validation rules on client and server sides.
Solution: Created shared validation utility functions and implemented comprehensive server-side validation.

Challenge 3: Concurrent Access Management

Problem: Multiple staff members accessing and modifying the same patient records.
Solution: Implemented optimistic locking and transaction isolation levels in PostgreSQL.
